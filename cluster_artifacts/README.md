# cluster_artifacts/ — manifests of files generated on the cluster

The preprocessing and inference steps generate **thousands of large files** under `~/run/work/`
on the Saarland HPC (`.dgl` graphs, extracted pockets, per-repeat docked poses). They are too
bulky to commit, so this folder **mirrors the cluster directory structure and lists the file
names** instead — as evidence of what was produced and for reproducibility. Regenerate any of it
with the `condor/*.sub` jobs (see `docs/RUN_COMMANDS.md`).

```text
~/run/work/                         (on the cluster)            -> manifest here
├── graphs/proto_train/  (712 .dgl) -> graphs generated by generate_graph.py
│         proto_test/    (75 .dgl)                                 work/graphs/*.dgl.list.txt
├── complex/proto_train/ (712 dirs) -> KarmaDock-layout complexes + 12 A pockets
│          proto_test/   (75 dirs)     (pre_processing.py)         work/complex/*.{complexes,pockets}.list.txt
├── kd_out/<pipeline>/proto_test/   -> docked poses per repeat (0/ 1/ 2/) + <re>.csv scores
│         (678 files each pipeline)    (ligand_docking.py)         work/kd_out/<pipeline>_proto_test.list.txt
├── ckpt/<pipeline>/                -> our training checkpoints + train_log.csv  work/ckpt/checkpoints.list.txt
└── logs/                           -> condor stdout/stderr per job             logs/logs.list.txt
```

See `COUNTS.txt` for a one-glance summary. Pocket extraction = protein residues within 12 A of the
ligand (matches the KarmaDock paper). ~15% of complexes are lost at graph generation to RDKit
ligand sanitisation (kekulize/valence) on raw PDBBind sdf — expected; documented in `docs/RUN_JOURNAL.md`.
